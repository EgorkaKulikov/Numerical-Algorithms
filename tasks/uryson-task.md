# Задание на разработку программы для численного эксперимента

## 0. Контекст и цель

В файле `main.tex` (раздел «Численные эксперименты», `\label{sec:numerical}`) описан вычислительный
эксперимент по приближённому решению нелинейных интегральных уравнений Урысона **первого** и **второго**
рода методом сплайн-коллокации в базисе **квадратичных минимальных $B_\varphi$-сплайнов** с
биортогональными проекционными функционалами $\theta_j$.

Таблицы в статье оставлены с пустыми числовыми ячейками. **Цель программы** — провести эксперимент и
а также убедиться (через health-checks), что реализация математически корректна.

Программа должна быть **одним самодостаточным файлом на Kotlin** с функцией `main`, **без внешних
зависимостей** (только Kotlin/Java stdlib). Вся линейная алгебра (решение СЛАУ, нормы, матрично-векторные
операции) реализуется вручную. Файл должен компилироваться и запускаться так:

```
kotlinc NumericalExperiment.kt -include-runtime -d ne.jar && java -jar ne.jar
```

или через `kotlin NumericalExperiment.kt` / Gradle-задачу проекта.

Никаких чтения/записи во внешние сервисы, сети, БД. Только консольный вывод и, опционально, запись
текстовых файлов с готовыми LaTeX-таблицами в рабочую директорию.

   аккуратное масштабирование, но не переход на произвольную точность.
3. **Подробная документация.** KDoc на каждом публичном типе и функции: что вычисляет, какие формулы из
   `main.tex` реализует (указывать номер/`\label` уравнения, например `// реализует (omega-explicit)`),
   единицы измерения параметров, предусловия и постусловия.
4. **Health-checks (самопроверки).** Перед основным расчётом программа обязана прогнать набор проверок
   (раздел 9). При провале любой критической проверки — печать диагностики и **аварийное завершение с
   ненулевым кодом возврата** (расчёт на заведомо неверной реализации запускать нельзя).
5. **Детерминированность.** Любая псевдослучайность (шум для уравнения I рода) — только через

6. **Воспроизводимость вывода.** Все числовые результаты печатаются в фиксированном формате (раздел 8),

7. **Чистый расчёт.** Никаких «подгонок» результатов: правые части модельных задач вычисляются из точного
   решения численно (квадратурой), решение находится методами из статьи, ошибки считаются относительно
   точного решения. Запрещено зашивать ожидаемые значения ошибок.

## 3. Порождающие системы (`GeneratingSystem`)

Реализовать тип, инкапсулирующий $\rho,\sigma$ и их производные до нужного порядка (минимум $\rho',\sigma'$;
для проверок и $\partial$-вычислений полезны $\rho'',\sigma''$):

- $\boldsymbol{\varphi}^{\mathcal B}(t)=(1,t,t^2)^T$ — полиномиальная (классические квадратичные $B$-сплайны);
- $\boldsymbol{\varphi}^{\mathcal H}(t)=(1,\sinh t,\cosh t)^T$ — гиперболическая;
- $\boldsymbol{\varphi}^{\mathcal T}(t)=(1,\sin t,\cos t)^T$ — тригонометрическая.

Дать осмысленные `name` («B», «H», «T») для подписей таблиц.

## 4. Базис минимальных сплайнов

Реализовать **точно по явным формулам** статьи (не приближённо):

1. Узлы $\varphi_j=\boldsymbol{\varphi}(x_j)$, $\varphi'_j=\boldsymbol{\varphi}'(x_j)$ для $j=-2,\dots,n+2$
   (на концах узлы кратности три: $x_{-2}=x_{-1}=x_0=a$, $x_n=x_{n+1}=x_{n+2}=b$).
2. Векторы $\mathbf{d}_j\in\mathbb{R}^3$ через тождество $\mathbf{d}_j^T\mathbf{x}=\det(\varphi_j,\varphi'_j,\mathbf{x})$
   (то есть $\mathbf{d}_j=\varphi_j\times\varphi'_j$). См. `(eq:dj-def)`/после `(wronsk)`.
3. Векторы $\mathbf{a}_j$ по формуле `(aj)`:
   $\mathbf{a}_j=\varphi_{j+1}-\dfrac{\mathbf{d}^T_{j+2}\varphi_{j+1}}{\mathbf{d}^T_{j+2}\varphi'_{j+1}}\varphi'_{j+1}$.
4. Сами $\omega_j(t)$ и их производные $\omega_j'(t)$ — по кусочной формуле `(omega-explicit)` на
   $[x_j,x_{j+1})$, $[x_{j+1},x_{j+2})$, $[x_{j+2},x_{j+3})$ и $0$ вне $[x_j,x_{j+3}]$. Производная нужна для
   матрицы Грама $R_h$ `(Rmat)`.

> **Важно для health-check.** Для $\boldsymbol{\varphi}^{\mathcal B}$ значения $\omega_j$ обязаны совпадать с
> классическими квадратичными $B$-сплайнами по формуле `(omegaB)`; для $\boldsymbol{\varphi}^{\mathcal H}$ —
> с `(omegaH)`. Эти явные формулы реализовать **отдельно** именно как эталон для сверки (раздел 9), а не
> переиспользовать общий код.

> **Важно для health-check.** Должно выполняться свойство разбиения единицы `(pou)`: $\sum_j\omega_j(t)\equiv1$ при $\varphi_0\equiv1$.

## 5. Проекционные функционалы $\theta_j$ и проектор $P_\theta$

Реализовать $\theta_j$ по формуле `(lam-form)`/`(pr_func)`: каждый $\theta_j$ — линейная комбинация значений
аргумента в узлах $x_j,x_{j+3}$ и в серединах сеточных интервалов
$\tfrac{x_j+x_{j+1}}2,\tfrac{x_{j+1}+x_{j+2}}2,\tfrac{x_{j+2}+x_{j+3}}2$. Коэффициенты $\beta_{j,k}$
выражаются через значения базисных функций в этих точках:
$$A=\omega_j(\tfrac{x_j+x_{j+1}}2),\quad C=\omega_j(\tfrac{x_{j+1}+x_{j+2}}2),\quad E=\omega_j(\tfrac{x_{j+2}+x_{j+3}}2),\quad B=\omega_j(x_{j+1}),\quad D=\omega_j(x_{j+2}),$$
$$K_1=C^2D-BCE-ADE-DE^2,\qquad K_2=-BCD+B^2E+AD^2,$$
и краевые функционалы $\theta_{-2},\theta_{-1},\theta_{n-2},\theta_{n-1}$ — по формуле `(pr_func)` из
`example.tex` / [prfunc]. Для полиномиального случая на равномерной сетке они обязаны редуцироваться к
классическим `(pr_func_b)` (отдельный health-check).

Проектор: $P_\theta g=\sum_j\theta_j(g)\,\omega_j$. Коэффициенты проекции — вектор $(\theta_j(g))_j$.
Биортогональность `(biorth)`: $\theta_i(\omega_j)=\delta_{i,j}$ — health-check.

## 6. Квадратуры и операторы

1. **Квадратура.** Реализовать составную формулу **Гаусса–Лежандра** (узлы/веса для фиксированного числа
   точек на подынтервал, например 5–10) на разбиении $[a,b]$, согласованном с сеткой $X$ (или более мелком).
   Это базовый инструмент для интегралов `(quad-int)`, весов $W_j=\int_a^b\omega_j$, матрицы $R_h$ `(Rmat)`,
   $L^2$-норм и невязок. Точность квадратуры должна быть заведомо выше точности аппроксимации (health-check
   на точном интегрировании полиномов/экспонент).
2. **Оператор Урысона** `(U-op)`: $(\mathcal{U}x)(t)=\int_a^b K(t,s,x(s))\,ds$, где $x(s)$ —
   вычисляется как сплайн $x_h$ (по коэффициентам) либо как функция (для построения правых частей).
3. **Производная Фреше** `(Uprime)`:
   $(\mathcal{U}'(x)\,h)(t)=\int_a^b \partial_u K(t,s,x(s))\,h(s)\,ds$. Нужна для якобиана $B(\mathbf{c})$
   `(Bdef)` в схеме I рода.

Ядра $K$ и $\partial_u K$ задаются аналитически для каждой модельной задачи (раздел 7).

## 7. Модельные задачи

Все на $[0,1]$. Правые части строятся **из точного решения численно** (подстановкой $x^\ast/x^\dagger$ в
оператор и интегрированием), что должно быть явно отражено в коде и логе.

### Уравнение Урысона II рода `(ury2)`: $x(t)=f(t)+\lambda\int_0^1 K(t,s,x(s))\,ds$

- **Задача A** `(eq:num-ury2A)`: $K(t,s,u)=\dfrac{1}{t+s+u}$, $\lambda=-1$, точное $x^\ast(t)=\dfrac{1}{t+1}$.
  Назначение: проверка **порядка сходимости** в нейтральной ситуации (решение не лежит в оболочке ни одной
  порождающей системы).
- **Задача B** `(eq:num-ury2B)`: $K(t,s,u)=e^{t-2s}u^3$, $\lambda=1$, $f(t)=e^{t+1}$, точное $x^\ast(t)=e^t$.
  Назначение: демонстрация **уменьшения константы** при $\boldsymbol{\varphi}^{\mathcal H}$
  (т.к. $e^t\in\operatorname{span}\{1,\sinh t,\cosh t\}$).

Конечномерная система — `(base-system-xi)`; решать **методом простой итерации** `(simple-iter)` до
сходимости по норме $\|\cdot\|_\infty$ коэффициентов (с порогом и лимитом итераций). Дополнительно
реализовать:
- **итерацию Sloan** `(sloan-iter)`/`(sloan-xi)` — однократная поправка $\widetilde x_h=f+\lambda\mathcal{U}x_h$;
- **модификацию Kulkarni** `(Kulk-eq)`/`(Kulk-system)`/`(xh-final)`;
- **сплайн-метод Nyström** `(Unystrom-xi)`/`(Nystrom-eq)` с весами $W_j$.

### Уравнение Урысона I рода `(ury1)`: $\int_0^1 K(t,s,x(s))\,ds=f^\dagger(t)$ (некорректная задача)

- **Задача C** `(eq:num-ury1C)`: $K(t,s,u)=\dfrac{1}{t+s+u}$, точное $x^\dagger(t)=\dfrac{1}{t+1}$.
- **Задача D** `(eq:num-ury1D)`: $K(t,s,u)=e^{-(t-s)^2}u^3$, точное $x^\dagger(t)=e^t$.

Схема — регуляризация Тихонова в сплайновом подпространстве `(tikh-disc)`:
$$\min_{\mathbf c}\;\bigl\|\Theta_h(\mathcal U x_h)-\Theta_h(f_h^\delta)\bigr\|_{\ell^2_h}^2+\alpha\,\mathbf c^TR_h\mathbf c.$$

- Веса $\ell^2_h$ `(lh2-norm)`: $w_j=\tfrac13(x_{j+3}-x_j)$; $W_h=\operatorname{diag}(w_j)$.
- Матрица Грама $R_h$ `(Rmat)` — симметричная, положительно определённая, полосная.
- Решение — **итерации Гаусса–Ньютона с тихоновской регуляризацией** `(coll-gn)`:
  $$(B(\mathbf c_k)^TW_hB(\mathbf c_k)+\alpha R_h)\,\Delta\mathbf c=-B(\mathbf c_k)^TW_h(\boldsymbol\Xi(\mathbf c_k)-\boldsymbol\theta(f_h^\delta))-\alpha R_h\mathbf c_k,$$
  где $[B(\mathbf c)]_{j,i}=\theta_j(\mathcal U'(x_h)\omega_i)$ `(Bdef)`.
- Шум: $f^\delta=f^\dagger+\xi_\delta$, $\|f^\delta-f^\dagger\|_{L^2}\le\delta$ (условие (VII)). Генерировать
  детерминированно (фиксированный `seed`), затем **отмасштабировать** возмущение под заданный $\delta$.
- Выбор $\alpha$ — **апостериорно по принципу Морозова** `(morozov)`:
  $\|\Theta_h(\mathcal U x_h^{\alpha,\delta})-\Theta_h(f_h^\delta)\|_{\ell^2_h}=\tau\,\bar\delta_h$, $\tau>1$
  (например $\tau=1.1$). Решение уравнения на $\alpha$ — монотонным поиском (бисекция по $\log\alpha$).

## 8. Эксперименты, метрики и вывод

**Сетки** `(net)`: $n\in\{8,16,32,64,128\}$.
- равномерная $X^{\mathrm u}$: $x_j=j/n$;
- квазиравномерная $X^{\mathrm q}$: гладкое возмущение равномерной с фиксированным (не зависящим от $n$)
  параметром локальной квазиравномерности (например, узлы $x_j=\Psi(j/n)$ с гладкой монотонной $\Psi$,
  $\Psi(0)=0,\Psi(1)=1$).

**Контрольная сетка** $\widehat X$: $|\widehat X|=100n+1$ равноотстоящих узлов.

**Метрики** (раздел `subsec:num-setup` статьи):
- $E_h=\max_{t\in\widehat X}|x^\ast(t)-x_h(t)|$ `(eq:num-Eh)` (для I рода вместо $x^\ast$ берётся $x^\dagger$);
- $E_h^{\mathrm{rel}}=E_h/\|x^\ast\|_{\infty,\widehat X}$;
- $p_h=\log_2(E_h/E_{h/2})$ `(eq:num-ph)` (в последней строке — прочерк);
- $C_h=E_h/h^{p}$, где $p$ — ожидаемый порядок ($p=3$ для квадратичной базовой схемы);
- для I рода дополнительно: дискретная невязка
  $\operatorname{res}_h=\|\Theta_h(\mathcal U x_h^{\alpha,\delta})-\Theta_h(f_h^\delta)\|_{\ell^2_h}$ и
  значение регуляризатора $\Omega(x_h^{\alpha,\delta})=\mathbf c^TR_h\mathbf c$.

**Соответствие таблицам `main.tex`** — программа должна напечатать ровно те величины, которые нужны для:
- `tab:num-ury2A-order` — Задача A, равномерная сетка, $\boldsymbol{\varphi}^{\mathcal B}$ vs минимальные:
  колонки $n,h,E_h,p_h$ (по два последних на каждый базис);
- `tab:num-ury2B-phi` — Задача B, три порождающие $\boldsymbol{\varphi}^{\mathcal B/H/T}$: $E_h,p_h,C_h$;
- `tab:num-ury2-methods` — Задача B, базис $\boldsymbol{\varphi}^{\mathcal H}$: базовая / Sloan / Kulkarni /
  Nyström, $E_h,p_h$;
- `tab:num-ury1C` — Задача C: строки по $(\delta,n,\alpha)$, колонки $E_h,E_h^{\mathrm{rel}},\operatorname{res}_h,\Omega$;
- `tab:num-ury1D` — Задача D: $\boldsymbol{\varphi}^{\mathcal B}$ vs $\boldsymbol{\varphi}^{\mathcal H}$ при
  фиксированном $\delta$: $E_h,\operatorname{res}_h$ и отношение $E_h^{\mathcal B}/E_h^{\mathcal H}$.

**Формат вывода.** Печатать (1) человекочитаемые таблицы в консоль и (2) опционально — фрагменты
LaTeX-строк (`a & b & ... \\`) в формате, совпадающем со столбцами соответствующих `\begin{tabular}` в
`main.tex` (числа — в нотации $m{,}mmm\!\cdot\!10^{p}$ или `1.234e-05`, на выбор, но единообразно), чтобы
их можно было вставить в таблицы напрямую. Также печатать значения $h$ для каждого $n$ (для сверки с
графой `$h$`).

## 9. Health-checks (самопроверки) — обязательны

Все проверки печатают имя, измеренную величину невязки и вердикт `OK/FAIL` с порогом. Критические провалы
останавливают программу с ненулевым кодом.

1. **Совпадение минимального сплайна с полиномиальным $B$-сплайном (ключевая, по запросу).**
   Для $\boldsymbol{\varphi}^{\mathcal B}$ на равномерной и на квазиравномерной сетке: для всех $j$ и набора
   точек $t$ значение $\omega_j(t)$, вычисленное по общей формуле `(omega-explicit)`, совпадает со значением
   по эталонной формуле `(omegaB)`. Аналогично производные. Порог $\le10^{-10}$.
2. **Совпадение с гиперболическим сплайном.** Для $\boldsymbol{\varphi}^{\mathcal H}$ общая формула `(omega-explicit)`
   совпадает с `(omegaH)`. Порог $\le10^{-10}$.
3. **Разбиение единицы `(pou)`.** $\bigl|\sum_j\omega_j(t)-1\bigr|\le10^{-10}$ для всех порождающих систем
   с $\varphi_0\equiv1$ на наборе точек.
4. **Биортогональность `(biorth)`.** $|\theta_i(\omega_j)-\delta_{i,j}|\le10^{-9}$ для всех $i,j$ и всех
   порождающих систем.
5. **Свойство проектора (лемма `lem:projector`).** Для $u_h\in\mathbb S_\varphi(X)$ выполнено $P_\theta u_h=u_h$:
   взять случайный вектор коэффициентов, спроецировать, сравнить. Порог $\le10^{-9}$.
6. **Точность на оболочке порождающей системы.** Для функции $g\in\operatorname{span}\{1,\rho,\sigma\}$
   (например, $g=\rho$) ошибка интерполяции/проекции $\max_t|g(t)-(P_\theta g)(t)|$ численно нулевая
   ($\le10^{-9}$). Для $\boldsymbol{\varphi}^{\mathcal H}$ это, в частности, $g(t)=e^t$.
7. **Редукция краевых $\theta_j$.** Для $\boldsymbol{\varphi}^{\mathcal B}$ на равномерной сетке коэффициенты
   $\theta_j$ совпадают с классическими `(pr_func_b)` (включая краевые и внутренние $1/14,10/7,-2/7$). Порог
   $\le10^{-10}$.
8. **Точность квадратуры.** Гаусс–Лежандр точно (до $\le10^{-12}$) интегрирует многочлены степени до
   $2m-1$ и с высокой точностью — $e^t$, $1/(t+1)$ на $[0,1]$ (сверка с аналитическим значением).
9. **Матрица Грама $R_h$ `(Rmat)`.** Симметрична ($\max|R-R^T|\le10^{-12}$), положительно определена
   (например, успешное разложение Холецкого), полосная (нули при $|i-j|>2$).
10. **Согласованность весов.** $\sum_j w_j=b-a$ (с учётом краевых $w$); проверка `(lh2-norm)`.
11. **Согласованность правой части.** Построив $f$ из точного решения и подставив само точное решение в
    дискретную невязку **без** регуляризации, убедиться, что невязка мала и убывает с измельчением сетки
    (контроль того, что правые части построены корректно).
12. **Эквивалентность шагов методов (sanity).** На грубой сетке проверить, что простая итерация для базовой
    схемы сходится, а одна итерация Гаусса–Ньютона уменьшает регуляризованный функционал `(tikh-disc)`.
13. **Симметрия/положительная определённость матрицы Гаусса–Ньютона** $B^TW_hB+\alpha R_h$ при $\alpha>0$.

Рекомендуется также «золотой» тест: для Задачи B с $\boldsymbol{\varphi}^{\mathcal H}$ ошибка на самой
мелкой сетке должна быть очень малой (решение лежит в оболочке) — это косвенно подтверждает корректность
всей цепочки; порог задать мягким и трактовать как предупреждение, а не критический отказ.

## 10. Рекомендуемая структура файла

1. Заголовочный KDoc: назначение, ссылка на `main.tex`/раздел, как запускать, что печатает.

4. **`GeneratingSystem`** и три экземпляра (B, H, T) с производными.
5. **Сетки**: построение `(net)` с тройными узлами, равномерная и квазиравномерная.

## 11. Параметры по умолчанию

- $[a,b]=[0,1]$; $n\in\{8,16,32,64,128\}$; контрольная сетка $100n+1$.
- Простая итерация: порог $10^{-12}$, лимит $10^4$ итераций.
- Гаусс–Ньютон: порог по $\|\Delta\mathbf c\|_\infty$ $10^{-10}$, лимит $\sim50$ итераций.
- Морозов: $\tau=1.1$, бисекция по $\log_{10}\alpha$ в диапазоне, например, $[10^{-12},10^{2}]$.
- Уровни шума для I рода: $\delta\in\{10^{-1},10^{-2},10^{-3},10^{-4},10^{-5}\}$ (Задача C), фиксированный
  $\delta$ для Задачи D; `seed` фиксирован и печатается.
- Гаусс–Лежандр: не менее 5 узлов на подынтервал (повышать при необходимости).

## 12. Критерии приёмки

1. Файл компилируется и запускается без внешних зависимостей; `main` отрабатывает до конца.
2. **Все критические health-checks (раздел 9) проходят**; их результаты напечатаны.
3. Для модельных задач напечатаны числа для **всех пяти таблиц** статьи в порядке и составе, описанном в
   разделе 8, с указанием $n$ и $h$.
4. Для II рода наблюдаемый $p_h\to3$ (базовая схема) и согласуется между полиномиальным и минимальными
   базисами; для Задачи B ошибка $\boldsymbol{\varphi}^{\mathcal H}$ систематически меньше, чем
   $\boldsymbol{\varphi}^{\mathcal B}$, при том же $p_h$.
5. Для I рода при согласованном $\delta,h,\alpha\to0$ ошибка $E_h$ убывает (устойчивость и сходимость к
   $x^\dagger$); для Задачи D ошибка $\boldsymbol{\varphi}^{\mathcal H}$ меньше, чем
   $\boldsymbol{\varphi}^{\mathcal B}$.
6. Код снабжён KDoc; в комментариях указаны соответствия формулам `main.tex` (по `\label`).
7. Результаты детерминированы (повторный запуск даёт те же числа).

> Примечание для исполнителя: при численных трудностях (плохая обусловленность $B^TW_hB+\alpha R_h$ при
> очень малых $\alpha$, медленная сходимость простой итерации при $q_h$ близком к 1) допускается аккуратное
> масштабирование, повышение числа узлов квадратуры и разумные ограничения диапазона $\alpha$ — но **без**
> подмены метода и **без** зашивания ожидаемых чисел. Любые такие меры документировать в комментариях.
